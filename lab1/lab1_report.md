# Лабораторна робота 1: Побудова B+ дерева та індекси

Реалізація індексної структури `BPlusTree` для зберігання прізвищ з хеш-ключами.

- `order = 4` — максимальна кількість дочірніх посилань у внутрішньому вузлі.
- `depth = 3` — максимальна глибина дерева (у мене корінь 0).
- Дані зберігаються в листках у форматі `(hash, name)`.

## 2) Опис структури дерева (БД) з прикладами коду

### 2.1 Вузол 

```python
class Node:
    def __init__(self, is_leaf=False):
        self.is_leaf = is_leaf
        self.keys = []
        self.children = []
        self.next_sibling = None
```

- `is_leaf` — ознака типу вузла.
- `keys`:
  - у внутрішніх вузлах: розділювальні ключі;
  - у листках: пари кключ-значення.
- `children` — дочірні вузли (використовується у внутрішніх вузлах).
- `next_sibling` — linked list листя.

### 2.2 Дерево 

```python
class BPlusTree:
    def __init__(self, order=4, depth=3):
        self.root = Node(is_leaf=True)
        self.order = order
        self.depth = depth
```

- `root` — кореневий вузол.
- `order` — порядок дерева.
- `depth` — обмеження максимальної глибини.

### 2.3 Основні функції: вставка, розщеплення, видалення

#### Вставка 

```python
def insert(self, name: str):
    key = self.hash_function(name)
    split_result = self.insert_helper(self.root, key, name)
```

Кроки:

1. Обчислюється хеш-ключ.

4. Якщо розщепився корінь — створюється новий корінь.

Ліва та права частини листка розходяться по дочірнім вузлам, третій по рахунку ключ залишається в новому внутрішньому вузлі. При розщепленні листків оновлюється next_sibling.

Після видалення виконується `_rebalance_delete(...)`:

- `borrow` з лівого сусіда;
- `borrow` з правого сусіда;
- `merge` із сусідом, якщо позичити неможливо;
- оновлення ключів батьківських вузлів;
- зменшення висоти, якщо у кореня лишився один нащадок.

Це забезпечує самобалансування B+ дерева після модифікацій.

### 2.4 Теоретична межа місткості

Для `order = 4`, `depth = 3` і `root` на рівні `0`:

$$
L_{\max} = 4^3 = 64
$$

де `L_max` — максимальна кількість листків.

У листку максимум `order - 1 = 3` записи, тому:

$$
N_{\max} = 64 \cdot 3 = 192
$$

Отже, теоретична максимальна кількість записів у дереві: **192**.

## 4) Схеми пошуку значень у дереві

### 4.1 Точковий пошук `search_exact(name)`

```
    A["Root"] --> B["Спуск за розділювальними ключами"]
    B --> C["Leaf"]
    C --> D["Порівняння (hash, name)"]
    D --> E["Знайдено / Не знайдено"]
```

### 4.2 `search_range(name, '>')`

```
    A["Root"] --> B["Leaf для опорного ключа"]
    B --> C["Перевірка значень у листку"]
    C --> D["Перехід по next_sibling"]
    D --> C
```

### 4.3 `search_range(name, '<')`

```
    A["Найлівіший Leaf"] --> B["Послідовний перегляд вправо"]
    B --> C{"key < target?"}
    C -->|Так| B
    C -->|Ні| D["Зупинка"]
```

### 4.4 Видалення  з перебалансуванням

```
    A["Root"] --> B["Пошук exact (hash, name) у листках"]
    B --> C["Видалення запису з leaf"]
    C --> D["rebalance: borrow/merge"]
    D --> E["Оновлення ключів батьків"]
    E --> F["За потреби shrink root"]
```
